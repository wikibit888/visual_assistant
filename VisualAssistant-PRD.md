# Visual Assistant v0.1 产品设计文档

> 文档状态：开发基线（决策人已冻结）　|　最后更新：2026-06-13（三支柱重构 + 瘦身版）
>
> **三项基线决策（已冻结）：**
> 1. **编排形态 = 工具调用 agent 循环（反应式，agentic 为主）**：planner LLM 每回合自主规划——直接出文本或发起 ≤N 轮工具调用。可彩排靠 planner 自由度收窄（§7.2）；rails mode 与 agentic 同代码、config 切换，仅作 M2 硬门后兜底。
> 2. **交付框定 = 单人 36h 现场 demo**：止损线、唯一牺牲位、彩排神圣性见 §10。
> 3. **产品定位 = 三场景助手**：**学习**（作业辅导，数学方程为其中一个 case；含坐姿守护）、**生活**（天气穿搭 + 日常）、**开放对话**（无场景预设，全交 LLM）。**开放对话 = 基座，学习/生活 = 对基座做减法的收窄。不设家长角色。**
>
> **设计脊梁——「planner 提议，确定性护栏裁决」：** agent 在"怎么帮"上自由；演示安全做成 **agent 改不动的确定性代码**（置信门控、澄清上限、姿态不抢话、视觉预算、loop 上限、可选答案护栏），**护栏在编排循环之外、不可被 planner 覆盖**。这是把 agentic 自主与 demo 确定性对冲掉的核心纪律。
>
> **已锁定的设计红线**：S4 物品鉴定删除；复述确认；批改四值 verdict；对讲机/自由对话双语音模式为正式特性；语音止损线（4h/7h/10h）；隐私基线（音视频不留存、姿态端侧）；答案拦截为可选护栏（非硬红线）。
>
> **评审定论**：经三轮评审，决策人在知情 agentic + 开放对话语义风险（§5.4）下，选定 agentic 为主 + 三场景；rails 为兜底；商业付费主体标"待定"，不在 36h 内结。

## 1. 产品定位与说明

### 1.1 一句话定位

**Visual Assistant 是一个"看得见、听得懂、自己会编排、还管得住自己"的桌面助手**——摄像头 + 麦克风实时多模态交互，由三个场景定义：**学习**（作业辅导 + 坐姿守护）、**生活**（天气穿搭 + 日常）、**开放对话**（什么都能聊）。每回合自主判断"要不要看、看什么、要不要查天气、怎么帮"，在确定性护栏内作答。

### 1.2 目标用户

| 用户 | 角色 | 核心诉求 |
|---|---|---|
| 桌前用户（以学生为锚，框架不设限） | 唯一主用户 | 学习/生活随手有人搭把手；不被打扰但被照看；任何临场问题都能接住；帮忙时不胡编、不抢话 |

> **不设家长角色。** 卖点是 **"一个自主但自律的 agent"**——能力开放，却在最该克制处（看不清就明说、不编造、不抢话、不失控）由确定性护栏锁死。"自律"在 v0.1 是**设计属性 + 演示看点**，非商业卖点（付费主体待定）。

### 1.3 产品立场（设计原则）

1. **帮忙，但不替你包办。** 作业默认"先引导、不代写"，但**不强制纯苏格拉底**——agent 自主决定给思路/提示/直接讲。"不直给答案"作为**可选护栏**（§3.1.5）按需开启。
2. **一直在看，但不打扰。** 坐姿守护后台静默；提醒由确定性护栏插入对话间隙，不由 agent 自发插话（§3.2.2）。
3. **先确认，再行动。** 视觉结果先复述确认；歧义先澄清（≤1 次）；看不清就明说、绝不编造（→ 诚实兜底，§5.4）。
4. **每一帧都花得值。** 视觉按需、结果文本化驻留；**快路径**（多数回合零工具，纯文本模型）是最大省钱杠杆。
5. **自主，但可控。** 三道确定性边界：loop 上限、护栏不可覆盖、超时即维持现场景。**非确定性只许出现在"语言与策略"，不许出现在"立场与时序"。**

### 1.4 产品形态：工具调用 agent

统一对话入口，编排核心是一个反应式 agent 循环（**反应式 = 被动等用户回合，agent 不自发开口**；唯一的"主动"是端侧姿态守护这条独立确定性护栏，非 agent 自发）：

```
用户语音 ─ 端侧 VAD/ASR ─→ asr.final
        ┌───────────────▼─────────────────────────────────┐
        │   Agent 编排循环（模块 A，反应式）                │
        │  感知 asr.final + 工作记忆 → 规划 planner LLM     │
        │  （温度0+结构化输出）：直接答(快路径) / ≤N 轮工具 │
        │  → dispatch: read_problem/check_draft/observe/    │
        │              weather.get/memory.* → 组合 → 候选回复 │
        └───────────────┬─────────────────────────────────┘
        ┌───────────────▼─────────────────────────────────┐
        │  确定性护栏（agent 不可覆盖）：置信门控·澄清上限· │
        │  loop 上限·视觉预算·（可选）答案护栏              │
        └───────────────┬─────────────────────────────────┘
                       │ tts.say → 模块 B 播放
   （模块 D 姿态守护 = 端侧并行，独立发 posture.alert，护栏层择机在 gap.open 插入）
```

**设计要点：**
- **路由即工具选择**：无独立意图分类器，路由 = planner 工具选择 + focus 维持。
- **模式由 planner 隐式推断（选择 A）**：详见 §7.2。无独立模式分类器、无显式 UI/唤词。
- **开放对话无预设、全交 LLM；学习/生活 = 基座的收窄**（§5.1）。多模态输入**仅摄像头帧，不做图片上传**。
- **粘滞/澄清/低置信 = 双轨**：planner 策略 + 确定性护栏（§7.4）。

### 1.5 范围声明

- **硬约束**：1 人 × 36h（AI coding agent 并行写独立模块；本人负责契约/设备调试/集成；agent 产出按 20% 计提 review）。
- **做**：单用户、单设备（带摄像头麦克风的笔记本）、中文语音；reactive 工具调用 agent；三场景。
- **不做**：主动发起式自主；多步长程规划；执行类工具（订外卖/读屏/控设备 → 一律"帮不上"）；物品鉴定（原 S4 删除，诉求由 `observe` + 开放对话吸收）；家长角色/家长摘要；成本看板；账号体系/多用户/长期存储/移动端原生 App；图片文件上传。
- **能力开放 ≠ 保障开放**：彩排保障覆盖学习/生活脚本路径 + 开放对话诚实兜底；翻车以"看不清/帮不上就明说"收口。
- **隐私基线**：音视频不留存、仅会话内内存中转；工作记忆仅内存不落盘；姿态全程端侧。

## 2. 三支柱总览与优先级

| 支柱 | 含义 | 包含能力 | agent 形态 | 优先级 |
|---|---|---|---|---|
| **学习** | 学习时的辅导与照看 | 作业辅导（数学方程为一个 case）+ 坐姿守护 | `read_problem`/`check_draft`（+ 可选答案护栏）+ 端侧 D 姿态 | **P0** |
| **生活** | 出门/日常帮手 | 天气穿搭（**唯一牺牲位**）+ 日常临场请求 | `observe` + `weather.get` | **P1** |
| **开放对话** | 什么都能聊，无预设 | 即兴题 / 问画面 / 闲聊 / 任意请求 | **无场景预设，全交 LLM**；自主决定调不调工具；诚实兜底 | **能力面** |

> **唯一牺牲位 = 天气穿搭**：进度落后 >2h（M2 结束判定）只能砍它，回收约 3h，不得改砍彩排或学习场景；砍后开放对话仍在（`observe` 不删）。
> **rails 边界**：rails 只锁学习/生活工具序；开放对话不套 rails，其止损靠"优雅收口"分支（§8）。
> **模式切换 = planner 隐式（选择 A）**：无显式 UI/唤词，每回合推断 `current_mode`（§7.2）。

## 3. 支柱一：学习（作业辅导 + 坐姿守护）

### 3.1 作业辅导（数学方程为彩排锚 case）

用户做作业遇到不会的题，指着练习册求助。**agent 自主判断**要不要识题、怎么帮（给思路/提示/直接讲解）、何时抓帧批改；**不强制纯苏格拉底**，引导只是可选风格。写完请 AI 检查草稿，定位错误位置与类型。数学方程是彩排锚 case，工具不限题型。
- **指代对齐**：整帧多模态理解 + 复述确认（兼盖工具往返延迟）+ 澄清≤1 次三层兜底。
- **识题一次、文本驻留**：识题成功缓存 `active_problem`，后续走零工具快路径；单题视觉 ≤3 次（识题1 + 批改1–2）。批改仅在用户明说"看看"且 agent 判定需要时抓帧。

#### 用户故事

| ID | 用户故事 | 验收标准 | 状态 |
|---|---|---|---|
| S1-01 | 我指着某题说"这道不会"，AI 读出我指的那道 | 自主 `read_problem`（整帧理解，不做指尖坐标）；**先复述确认**；**澄清≤1**（确认体话术，第二次请用户报题号）；**置信低→护栏拦截，明说看不清，绝不猜题**；演示用整本练习册预实测 | 计划 |
| S1-02 | 卡住时我希望 AI 帮我理解，而不是只甩答案 | 自主选帮法（方向/提示/讲解）；**提示粒度阶梯**（方向级→操作级→示范级）为可用风格非强制序列；开启答案护栏时仍以"先引导"为默认 | 计划 |
| S1-03 | 我写完说"你看看"，AI 指出第一处错误 | 自主 `check_draft`；只定位错误行与类型，不报正确答案；**四值 verdict**（§7.7 契约三）；**非高置信不播报错误**（置信门控），改请用户念该行；演示主路径用工整字迹 + 预埋移项符号错误；**M5 必须补"非标准正确解法"专测**（防把对的判成错）；预授权口述批改降级剧本 | 计划 |
| S1-05 | 我说"太难了"时 AI 切鼓励话术、降粒度 | 安抚分支 + 粒度下沉作 planner 策略 | 计划 |
| S1-06 | 结束时 AI 给一句口头小结 | 面向用户的口头小结（做了几道、卡点、提醒坐姿几次）；**零状态机**：工作记忆 + 对话历史一次性交文本模型总结 | 计划 |

> 早期细化项（同类错误主动点破 / 错题归类 / 多题节奏；评委挑战并入开放对话）已删或并入。

#### 3.1.5 答案护栏（可选，非硬红线，demo 默认关）

去家长后"不报答案"降为可选护栏，配置开关决定。启用时：① 识题时模型顺带解出 `solved_answer`（不播报、不入明文日志）；② 候选回复出站前**正则组合拦截**（触发词 + 数值组合命中才触发→替换为追问话术，纯字符串、无 LLM、无延迟）；③ 误杀专测**含生活语境数字**（温度/年龄/楼层）；④ 拦截层在循环外（契约九）。可选增强（不默认）：方程题型加 sympy 本地二次校验（与 `solved_answer` 不一致→退回纯引导，~2h，仅 learning 启用）。

### 3.2 坐姿守护

学习中端侧持续监测坐姿，检测到驼背且持续后由确定性护栏在对话间隙温和提醒；提醒次数计入口头小结。**姿态检测 100% 端侧**（MediaPipe Pose 算颈/背夹角 + 头部位置），话术预置模板随机抽取不调 LLM——本功能云成本恒为 0。

| ID | 用户故事 | 验收标准 | 状态 |
|---|---|---|---|
| S2-01 | 驼背时收到温和提醒，但低头写字不误判 | **双条件触发**（胸椎后凸角度 + 头部位置/与桌面距离）而非单颈角；持续阈值 >90–120s（明显超过演示中任何一次低头写字）；固定阈值 + 单级话术随机抽取，不经 agent/LLM | 计划 |
| S2-02 | 提醒不打断我的解题思路 | 只在 `gap.open` 窗口播出（契约四）；**可丢弃不排队**；同类 60s 冷却；持续 20s 无间隙允许句界插入；对讲机模式间隙判定确定性（松键 + TTS 结束 ≥1s） | 计划 |

> **§3.2.2 姿态守护的权威约束（全文引用此处）：**
> - **提醒永不是 planner 动作**：来自端侧 D + 确定性间隙仲裁，**不是 LLM 自发开口**。模块 D 100% 端侧、只发 `posture.alert`、绝不出声、绝不入 agent loop。M0 钉死。
> - **作用域 = 学习语境**：`posture.alert` **仅在 `current_mode==learning` 或 `active_problem!=null` 时放行**。⚠️ 关键修法（根因级）：放行条件并入 `active_problem!=null`，把放行从"易被 planner 隐式抖动"的 `current_mode` flag **解耦**到更稳的 `active_problem` 状态——防止 mode 误判把 P0 坐姿提醒**静默吞掉**（评委台上无感知、收尾小结才暴露）。D 仍只读这两个 flag、不读 planner，红线不破。
> - **演示口径**：§9 坐姿提醒由**演示者主动做明显驼背动作触发（导演触发）**，不依赖自动触发，避开"低头写字时段被误判"的时机重合。

> 已删除：提醒升级（多级语气）/ 起身建议 / 趋势周报。

## 4. 支柱二：生活（天气穿搭 + 日常）

### 4.1 天气穿搭（唯一牺牲位）

用户出门前对镜头问"这样穿行吗"。agent 自主编排 `weather.get` + `observe`（单帧穿搭识别），给**具体到行动**的建议（加/减某件、带伞），不播报天气。
- 天气走 **Open-Meteo（无 key）** + 城市/小时缓存 + 写死兜底；自动定位（`navigator.geolocation`→`weather.get`），失败/超时/拒绝**静默回落上海**，绝不阻塞。穿搭识别 `observe` ≤1 帧，不立准确率验收（"建议说得通"即过）。

| ID | 用户故事 | 验收标准 | 状态 |
|---|---|---|---|
| S3-01 | 我问"这样穿出门行吗"，AI 结合天气和穿着给结论 | 结论含可执行动作；单帧识别；天气 Open-Meteo + 自动定位（失败回落上海）+ 断网回落写死兜底（断网可演、兜底必测） | 计划 |

> **风险记录**：桌面 Chrome 定位常基于 IP，可能返回机房/VPN 出口城市（成功但错误）。演示主路径默认 `MOCK_WEATHER` 或不念具体城市/温度，只给行动建议。

### 4.2 日常临场请求

天气穿搭之外的日常请求由同一 loop 处理；能用工具组合答的就答，用不上工具或越界的**统一诚实兜底**（§5.4），不编造。无独立脚本。

## 5. 支柱三：开放对话（无预设、全交 LLM）

### 5.1 定位 ·"只建两样"· 基座/收窄（权威段）

agentic 的最大产品价值：**同一 loop 处理任意临场请求，不做任何场景预设——所有对话直接交给 LLM**，而不是落入"未识别意图"的死分支。

> **开放对话 = 三支柱的基座，学习/生活 = 对它做减法的收窄**（全文引用此处）。一个 agent loop + 多模态 I/O 基座，上叠两套约束 profile，不是三条独立流水线；rails = 往基座注入更死的约束。内容不做工程化（纯 LLM，接受幻觉风险，§5.4）。开放模式真正要建的只有两样：
> 1. **多模态 = 仅摄像头帧**（按需抓帧 / `observe` / `read_problem`），不做图片上传。
> 2. **模式切换 = planner 隐式（选择 A）**：无显式 UI/唤词，每回合判 `current_mode`（§7.2）。

### 5.2 开放模式边界处理（要处理的就这些）

| # | 类别 | 触发 | 处理 |
|---|---|---|---|
| F-1 | 帧·指代 | "这个/这道/这张" | planner 抓帧；抓不到有意义内容 → "我没看清，指一下/挪近点" |
| F-2 | 帧·低置信 | 模糊/逆光/遮挡 | 置信门控 → "看不清，口述或重来"，绝不编造 |
| F-3 | 帧·多目标 | 镜像 / 背景有人或杂物 | 取主体或反问，不瞎描述 |
| F-4 | 帧·频次 | 评委连续举东西 | 视觉预算护栏封顶 |
| M-1 | 切换·推断 | 每回合 | planner 输出 `current_mode∈{open,learning,life}`，sticky、强信号才变 |
| M-2 | 切换·误判 | 把"穿搭"当"作业"等 | **无硬护栏**（选择 A 代价）；靠复述确认 + 澄清≤1 兜，接受的 agentic 风险 |
| M-3 | 切换·多意图 | 一句话夹两请求 | 取主意图；loop_cap 防一轮塞多套工具；次意图口头记/下轮 |
| M-4 | 切换·退出 | 换话题/任务完成 | 清 `active_problem` 切回 open；否则粘滞兜底 |
| M-5 | 坐姿放行 | mode 变化 | 仅 `current_mode==learning` 或 `active_problem!=null` 放行 `posture.alert`（§3.2.2，防 mode 抖动静默吞 P0） |
| C-1 | 能力·越界 | 订外卖/读屏/控设备 | 无工具 → "帮不上"，不假装（§1.5） |
| C-2 | 能力·事实幻觉 | 珠峰多高之类 | **不管**（纯 LLM，§5.4 明示接受） |
| X-1 | 控制·打断 | 处理帧时被打断 | stop 立即停 + 清队列 + ack（契约二） |

### 5.3 用户故事

| ID | 用户故事 | 验收标准 | 状态 |
|---|---|---|---|
| S-OPEN-01 | 我当场写一题 / 问画面里的东西 / 随口聊，AI 都能接住 | 全开放、无脚本；自主决定抓帧/调工具；视觉低置信→诚实兜底；越界→"帮不上"；主持话术坦诚（"即兴不保证，敢不敢试"）；彩排用陌生字迹/物体实测 ≥3 次，**含双意图诱导句、防误锁 learning 各一条** | 计划 |

### 5.4 保障合同（诚实兜底的权威定义，全文引用此处）

> **开放对话的"保障"靠诚实兜底，不靠正确率。** 由 LLM 直接负责内容，**没有场景安全网**：护栏只覆盖"看清没看清"（视觉低置信→兜底）+ loop/视觉预算；**覆盖不了"看清了但讲错"**（纯文本事实、几何/超纲题、生活常识），LLM 可能高置信讲错且不经护栏——**该支柱的固有属性，非疏漏**。安全合同 = **① 看不清/帮不上 → 诚实说明；② 期望管理话术；③ 不承诺执行类能力**。**演示定位"接得住任意请求"，而非"答得对"。此处无 mock 退路**（内容错误在 LLM 语义层，评委的题不在 fixture 里），知情接受。

## 7. 系统架构与端云协同

### 7.1 总体架构与模块定位

技术栈：**Chrome 页面**（getUserMedia、内建 AEC、MediaPipe JS、vad-web）+ **Python/FastAPI/单 WebSocket 后端**。**逻辑 5 模块、物理 2 进程**，不用 Electron。

| 模块 | 角色 | 物理位置 |
|---|---|---|
| **A 编排核心** | Agent 编排器：工具调用循环、工作记忆、dispatch、**确定性护栏**、间隙仲裁、turn_id、rails 切换 | `server/a_core/` |
| **B 语音 I/O** | VAD/PTT/播放队列（前端）+ 云 ASR/TTS 适配器（后端） | `web/src/modules/b_voice.js` + `server/b_voice/` |
| **C 视觉服务** | `vision.*` 工具实现；重试与预算 | `server/c_vision/` |
| **D 姿态守护** | 纯端侧，只发 `posture.alert`，不出声、不入 agent loop（§3.2.2） | `web/src/modules/d_posture.js` |
| **E 技能/提示词库** | 引导/穿搭/口头小结/开放对话 system prompt；提示词权属归 E | `server/e_skills/` |

**铁律**：跨模块只走总线消息（契约一），禁互相 import 内部对象；`gap.open` 只由 A 广播，`tts.say/stop` 只发 B，D 只输出 `posture.alert`；**扬声器每个字都经 A 的确定性护栏**；planner 不得绕过护栏直产 TTS，护栏不得被提示词关闭。

### 7.2 Agent 编排循环（契约八核心）

每回合（`asr.final` 到达）触发一次循环（伪码见 §1.4）。要点：
- **快路径优先**：planner 被提示"能直接答就别调工具"；识题后回合尽量零工具。
- **自由度收窄（可彩排的关键）**：deepseek-chat **温度 0 + 结构化输出（JSON schema）+ 工具白名单**；`kind∈{answer,tool_calls,clarify}`，`tools` 取自 §7.3 枚举。**自由度只留在措辞，不留在 `kind` 与工具选择**。M5 一致性专测指标 = **工具调用序列一致**（非答案逐字一致）。
- **模式字段（隐式切换，选择 A）**：planner 结构化输出含 `mode∈{open,learning,life}`，写入 `current_mode`（**sticky，强信号才变**，防抖动）；mode 选定生效工具子集与约束 profile，并门控坐姿放行（§3.2.2）。**无硬护栏防误判**——靠复述确认 + 澄清≤1 + 粘滞兜底（接受的 agentic 风险，§5.2 M-2）。
- **rails 与 agentic 同代码、config 切换**：dispatch 前置可注入 `forced_tool_sequence`（学习场景 `[read_problem, guide, check_draft]`——`guide` 为强制 answer 节点非工具，M0 须把序定义为"工具序 + answer 节点"混合）、`max_tool_rounds:2→0`；**M0 即把序做成可注入 hook（+0.5h），M2 切换零成本**。开放对话不套 rails。
- **loop 上限**：每回合工具往返 ≤ `max_tool_rounds`（默认 2）；触顶走 FALLBACK_TEXT。
- **超时即维持现场景**：planner 超时（默认 800ms 软超时）→ 维持 focus、走降级话术，绝不阻塞首响。

### 7.3 工具注册表（契约八的一部分，权属归 A）

| 工具 | 入参 | 出参 | MOCK |
|---|---|---|---|
| `vision.read_problem` | （即刻抓帧） | `{problem_text, confidence}` | `MOCK_VISION=1` 读 fixture |
| `vision.check_draft` | （即刻抓帧） | 四值 verdict `{verdict, error_line?, error_type?, confidence}` | fixture |
| `vision.observe` | `{hint?}` | `{description, confidence}`（穿搭/即兴物体通用） | fixture |
| `weather.get` | `{lat, lon}` | `{summary, temp, precip, …}` | `MOCK_WEATHER=1` 返回兜底 |
| `memory.note` / `memory.recall` | `{key, value?}` | 工作记忆读写（仅内存） | 进程内 |

> "说话"不是工具，是循环终点；`posture.*` 不进工具表（姿态是 push，不可被 agent 拉成自由动作）；无执行类工具（越界请求 → "帮不上"）。

### 7.4 确定性护栏（agent 不可覆盖）

| 护栏 | 作用 | 契约 |
|---|---|---|
| 置信门控 | 视觉 `confidence` 低 → 不播报错误/不开讲，改请用户念/口述 | 三 |
| 澄清上限 | 每 focus 澄清 ≤1 次 | 八 |
| loop 上限 | 每回合工具往返 ≤ `max_tool_rounds` | 八 |
| 视觉预算 | 单会话/单题视觉上限（识题1+批改≤2） | 八/C8 |
| 粘滞兜底 | planner 超时/失败 → 维持 focus | 八 |
| 姿态不抢话 | 提醒仅在 `gap.open` 插入、可丢弃不排队；放行门控见 §3.2.2 | 四 |
| 答案护栏（可选） | 出站前正则组合拦截 `solved_answer`，按开关启用 | 九 |

> **护栏盲区（权威说明，全文引用此处）**：以上护栏只守"看清没看清 / 时序 / 失控"，**守不住"看清了但 LLM 讲错"**（语义层错误）。开放对话最突出（§5.4），知情接受。**护栏是地板，planner 是天花板**；演示安全只依赖地板。

### 7.5 语音交互体验目标

| 指标 | 目标 | 手段 |
|---|---|---|
| 首响延迟 | 文本回合 ≤1.5s；工具回合首句填充语 ≤1.5s，实质回答尽力而为 | 快路径=1 次 LLM 直答；工具回合用复述/填充语盖往返；按句 TTS 首句先播；planner 超时 800ms 维持现场景 |
| 工具往返 | 每回合 ≤ `max_tool_rounds`（默认 2） | loop 上限；多数回合 0 工具 |
| 打断/自激 | 双模式正式特性 | **开场默认对讲机**（按键说话/打断，间隙确定性）；**自由对话（VAD）** 作高光（AEC + SPEAKING ≥300ms 连续人声判打断）。**自激加固**：AI_SPEAKING 期间**半双工硬 gate 麦克风**（暂停采音），消灭自激链；代价 = 该期间不支持插嘴打断 → **"演打断"用对讲机按键演，不在自由对话里演**（防手贱触发自激） |
| 轮次判定 | 不抢话、不冷场 | 纯 VAD 静默阈值；对讲机由按键确定 |

> 语音端到端延迟与打断自激是语音链路第一风险（唯一无 mock 退路），半双工 gate + 开场对讲机是其唯一加固点（§10 止损线 4h/7h/10h）。

## 8. 降级与稳定性策略（演示保障）

| 风险 | 降级方案 |
|---|---|
| agent 循环乱跑（agentic 主风险） | **M2 硬门 → rails mode**（config 切换、同代码）：注入确定性工具序，agent 只填语言；M4 须彩排一次 |
| rails 接不住开放对话 | 超出工具序的请求 → **统一"优雅收口"分支**（"帮不上，回到作业/穿搭"），不卡死、不死分支 |
| agent 失控循环/超调用 | loop 上限 + 视觉预算（C8）；触顶 FALLBACK_TEXT |
| 语音延迟/打断自激（第一技术风险） | AI_SPEAKING 半双工 gate；开场默认对讲机；间隙自动切定时开窗；自由对话作高光、翻车即切回 |
| **坐姿 P0 静默漏报**（mode 误判吞 alert） | 放行并入 `active_problem!=null`（§3.2.2，根因级解耦）；§9 导演触发前钉强 learning 信号 |
| 现场网络弱 | 链路预热缓存；学习备用题预识别；天气写死兜底；TTS 失败回退字幕 |
| 摄像头/麦克风权限被拒 | 文案重试；终极降级纯文字模式 |
| 识题失败 / 批改低置信 | 澄清≤1 → 报题号/口述；verdict 门控不播报错误、改请念该行；M5 专测非标准正确解法；预授权口述批改 |
| 开放对话讲错（语义层、无 mock） | 诚实兜底 + 期望管理话术；演示定位"接得住"非"答得对"（§5.4） |

## 9. 演示动线（5–6 分钟 + 备选挑战）

1. **开场（30s）**：唤起助手，一句话定位（"会看会听、自己编排、又管得住自己——学习、生活、随便聊都行"）+ 介绍交互模式（默认对讲机，自由对话作高光；打断用对讲机按键演）。
2. **学习段（2–3min）**：指题求助 → 复述确认 → agent 自主帮解（多数回合零工具快路径）→ 写完请批改（预埋移项符号错误）→ **导演触发**：演示者先说一句强 learning 信号再做明显驼背 → 间隙提醒（展示"不打扰"且非 agent 自发）。
3. **生活段（1min）**：说"我要出门了" → 照镜头问穿搭（自主编排 `observe` + `weather.get`，默认不念具体城市/温度）。
4. **开放对话段（agentic 高光，1min）**：邀评委当场手写题 / 问画面物体 / 随口聊（"即兴不保证，敢不敢试"）。展示 agent 不落死分支、接得住任意请求；看不清/帮不上即诚实说明。
5. **收尾（30s）**：一句口头小结（做了什么、提醒坐姿几次）。

## 10. 里程碑（1 人 × 36h，多 agent 并行 + agentic 止损线）

> **风险对冲（agentic 知情风险）**：agentic 非确定性与可彩排冲突、M5「连续 3 次无致命失误」难度高。对冲：① M2 硬门 + rails（同代码切换）；② 护栏全确定性，非确定性圈在"语言/策略"层；③ 天气穿搭为唯一牺牲位；④ 自由度收窄（温度0+结构化+白名单）使 agentic 可彩排。

| 阶段 | 时间 | 本人（串行） | AI agent（并行） | 冒烟点 / 止损线 |
|---|---|---|---|---|
| M0 契约 | 0–2h | 契约一~十 + 工具注册表 + planner prompt + fixture；**`forced_tool_sequence` 可注入 hook（含 answer 节点）**；冻结契约八/十 | — | 契约入库 |
| M1 语音链路 | 2–12h | 真机调 VAD/流式 ASR/按句 TTS/**半双工 gate**/打断；默认对讲机优先调稳 | A=编排器骨架（loop+dispatch+护栏+收窄）/ C=视觉工具 / D=姿态（双条件）/ E=技能（每 2h 验收） | 4h 裸链路不通→全程对讲机；7h 自激压不住→锁半双工+对讲机；**12h 硬墙**：锁定语音模式禁碰底层 |
| M2 主场景合体 | 12–20h | mock 跑通 agent 循环（识题→帮解→批改）→ 真工具 + 置信门控（+ 可选答案护栏/sympy）；**坐姿放行并入 active_problem** | E 提示词、护栏测试、planner 调参 | 学习场景跑通；**agentic 硬门**：循环 3 次干跑不稳→切 rails；落后 >2h → 砍天气穿搭 |
| M3 生活+开放 | 20–27h | 生活（observe+weather）；开放对话（全交 LLM + 诚实兜底 + 优雅收口）；坐姿导演触发联调 | 生活编排、observe fixture | 生活段 + 开放对话段跑通 |
| M4 加固 | 27–31h | 降级全量：rails 切换、对讲机/半双工、口述/断网兜底、答案护栏误杀专测（含生活语境数字）、loop/失控专测、开放兜底专测 | — | 任一降级路径可演；rails 彩排过一次 |

## 11. 成功指标（验收口径）

- **学习·视觉**：指题命中率 ≥80%（整本练习册，数学方程 case）；澄清 ≤1；批改在预埋错误上定位 100%；**非标准正确解法不误判**；模糊草稿输出 unreadable 而非编造。
- **学习·坐姿**：双条件 + 长阈值，演示中低头写字零误报；**放行不被 mode 抖动吞**（导演触发可稳定演一次）；提醒不抢话、可丢弃。
- **生活**：穿搭结论含可执行动作；断网/定位失败兜底可演。
- **开放对话**：**不编造为硬指标**（看不清/帮不上即明说）；正确率不验收；agent 不落死分支、接得住任意请求。
- **编排自主性**：自主决定识题/帮解/批改时机，无写死序列；同一动线连续 3 次**工具调用序列一致**、不乱跑、不超 loop 上限。
- **语音自然度**：文本回合首响 ≤1.5s；工具回合首句填充 ≤1.5s；**半双工下无自激**；对讲机无缝；自由对话高光可演。
- **成本**：坐姿云调用 = 0；学习单题视觉 ≤3 帧；天气穿搭 ≤1 帧；多数回合走 0 工具快路径。
